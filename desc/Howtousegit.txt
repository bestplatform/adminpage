git의 초기화
  1. user.email과 user.name를 등록한다.
      전역 변수로 선언할때(--global), 지역 변수로 선언할때 (--global을 제거)
      ex)$ git config --global user.email "<이메일 주소>"
         $ git config --global user.name "<이름>"

  2. git의 초기 설정 자동완성
      $ git init

  3. 파일이 저장될 리모트 저장소 지정
      $ git remote add origin http://github.com/imtaekh/my_app.git
      온라인 저장소를 설정합니다.(origin을 해당 주소로 설정)

git 서버에 저장
  1. 변경된 내용을 컴퓨터 내부 git 저장소에 저장
      $ git add .(.은 한칸띄어서)
  2. 변경된 내용의 설명을 작성한다.
      $ git commit -m "<commit message>"
  3. 리모트(서버)에 저장한다.
      $ git push -u origin master

git 서버의 내용을 내컴퓨터에 저장
  git pull
  git clone + <해당 github주소>를 하면 해당 온라인 저장소가 내 컴퓨터로 다운로드 됩니다

모든 commit들과 디테일을 보여줍니다.
git log

commit id의 일부와 commit message만 보여줍니다.
git log --oneline

해당 commit으로 돌아가기
git reset --hard <commit id>

git push (리모트 저장소) (리모트 브랜치)
새로 생성한 로컬 브랜치에서 파일을 만들고 commit을 한 다음 아래와 같이 push를 하면
git push origin utility
(리모트 저장소)에 (리모트 브랜치)를 생성하고 현재의 로컬 브랜치와 동기화를 시킨다.
리모트 브랜치 상태보기

git remote
git remote show (리모트 저장소)
git remote show origin
위와 같이 명령어를 입력하면
$ git remote show origin

  Push  URL: git@github.com:mylko72/FEDNote.git
  HEAD branch: master
  Remote branches:
    gh-pages tracked
    master   tracked
  Local branches configured for 'git pull':
    gh-pages merges with remote gh-pages
    master   merges with remote master
  Local refs configured for 'git push':
    gh-pages pushes to gh-pages (up to date)

리모트 브랜치와 로컬 브랜치의 관계를 상세히 볼수 있다.
다시 말해 어떤 로컬 저장소가 리모트 저장소와 track 상태에 있는지 확인할 수 있다.

브랜치 삭제
git branch -D (브랜치)
git push (리모트 저장소의 별명) :(리모트 브랜치)
로컬 브랜치를 삭제하려면 아래 명령어를 사용한다.
$ git branch -D utility
Deleted branch utility (was e7f33f9).
리모트 브랜치를 삭제하려면 아래와 같이 입력하면 리모트 브랜치를 삭제할 수 있다.
$ git push origin :utility
To git@github.com:mylko72/FEDNote.git
 - [deleted]         utility
또는 아래와 같이 --delete 옵션을 사용하여 삭제한다.
$ git push origin --delete utility
